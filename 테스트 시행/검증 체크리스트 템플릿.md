## 1. 기본 준비
- [ ] `최근 수정 파일.md` 내용을 확인했다.
- [ ] `실행환경\Run-All.cmd` 또는 `Run-Server.cmd` / `Run-App.cmd` 실행 경로를 확인했다.
- [ ] 테스트는 NAS/Git 반영 전에 먼저 진행한다.

## 2. 서버 기동 확인
- [ ] 로컬 테스트 서버가 정상 기동된다.
- [ ] `/healthz` 응답이 정상이다.
- [ ] 데스크톱 앱이 로컬 테스트 서버 URL을 바라본다.

## 3. 이번 수정사항 확인
- [ ] 이번 수정 화면/기능을 직접 확인했다.
- [ ] 변경 영향이 있는 인접 화면도 함께 확인했다.
- [ ] 오류 메시지/예외/레이아웃 깨짐이 없는지 확인했다.
- [ ] 시작 시 업데이트 자기검사/무결성 점검 메시지가 비정상적으로 반복되지 않는다.

## 4. 저장/동기화/출력 확인
- [ ] 저장이 정상 동작한다.
- [ ] 자동저장/전환/닫기 동작을 확인했다.
- [ ] dirty 데이터가 남은 상태에서 업데이트가 차단되는지 확인했다.
- [ ] pending sync outbox가 남은 상태에서도 업데이트가 차단되는지 확인했다.
- [ ] 필요한 경우 출력/미리보기까지 확인했다.

## 5. 데스크톱 버전 확인
- [ ] 데스크톱 앱 수정이 포함된 경우 `D:\거래플랜\Desktop\거래플랜.Desktop.App\거래플랜.Desktop.App.csproj` 의 `Version` / `FileVersion` 을 올렸다.
- [ ] 반영 예정 버전이 현재 stable/live 배포본보다 높다.
- [ ] live 사전 점검(`Invoke-LiveReleaseReadinessCheck.ps1 -Mode Pre`)을 통과했다.
- [ ] 설치 패키지 내 `Updater\거래플랜.Updater.exe`, `appsettings.json` 포함 여부를 사후 점검에서 확인했다.
- [ ] 필요한 경우 `준비-다중PC-검증.ps1` / `다중PC-검증 시나리오.md` 기준 다중 PC 검증을 수행했다.
- [ ] `Invoke-SyncRecoveryCheck.ps1` 리포트가 PASS인지 확인했다.

## 6. 운영 안정화 스모크
- [ ] 관리자 계정으로 동기화 진단/무결성 리포트/업데이트 준비 상태를 확인했다.
- [ ] 권한 계정(예: USENET / ITWORLD / YEONSU)으로 로그인해 본인 화면 범위와 저장 권한이 맞는지 확인했다.
- [ ] 가능하면 `Invoke-AccountScopeRegressionCheck.ps1` 또는 `tasks\Run-OptionalAccountScopeRegression.ps1` 결과를 확인했다.
- [ ] 시작 시 무결성 점검 메시지가 비정상 반복되지 않고, 오래 멈춘 outbox가 있으면 자동 복구되는지 확인했다.
- [ ] `%LOCALAPPDATA%\거래플랜\logs\yyyyMMdd.log` 의 `[PERF]` 로그를 확인해 시작/화면전환 지연 구간이 비정상적으로 길지 않은지 확인했다.
- [ ] 첨부/증빙 파일을 일부러 이동하거나 삭제했을 때 무결성 리포트에 누락 이슈가 잡히는지 확인했다.

## 7. 반영 결정
- [ ] 문제 없음 → NAS 반영 가능
- [ ] 문제 없음 → Git 반영 가능
- [ ] 이슈 있음 → NAS/Git 반영 보류
