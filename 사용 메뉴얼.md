## 1. 프로그램 개요

거래플랜은 다음 업무를 한 프로그램에서 처리하는 데스크톱 ERP 도구입니다.

- 거래처 등록 / 수정 / 삭제 / 조회
- 품목 / 재고 등록 및 조회
- 판매 / 구매 전표 작성
- 수금 / 지급 처리
- 렌탈 거래처 등록, 렌탈 청구관리, 렌탈 자산 관리
- 납품내역 조회
- 자료 기간별 집계
- 동기화 / 백업 / 휴지통 관리

업무 흐름은 보통 아래 순서로 사용합니다.

1. 거래처 등록
2. 품목 등록
3. 판매 / 구매 전표 작성
4. 수금 / 지급 입력
5. 렌탈이 있으면 렌탈 거래처 등록 / 렌탈 청구관리
6. 인쇄 / 자료집계 / 백업 / 동기화

## 2. 시작하기

### 실행 파일
일반적으로 아래 경로의 실행 파일을 사용합니다.

- `C:\Program Files (x86)\tradeplan\거래플랜.exe`

### 처음 실행 후 확인할 것
1. 프로그램이 열리면 로그인 창이 표시됩니다.
2. 로그인 후 `대시보드 알림` 창이 먼저 뜰 수 있습니다.
3. 안내 문구를 확인한 뒤 `확인`을 누르면 메인화면으로 이동합니다.

> 팁  
> 로그인 직후 안내 창이 떠 있는 동안에는 메인 버튼이 눌리지 않는 것처럼 보일 수 있습니다. 팝업의 `확인`을 누르면 메인화면으로 이동해 계속 작업할 수 있습니다.

## 3. 로그인 / 로그아웃

## 로그인
### 화면 이름
- `거래플랜 - 로그인`

### 입력 항목
- `아이디`
- `비밀번호`
- `아이디 저장`
- `비밀번호 저장`
- `로그인`
- `오프라인 모드로 시작` (환경에 따라 보일 수 있음)

### 사용 순서
1. `아이디` 입력
2. `비밀번호` 입력
3. 필요 시 `아이디 저장`, `비밀번호 저장` 체크
4. `로그인` 클릭
5. 로그인 후 `대시보드 알림` 창이 뜨면 `확인`

## 로그아웃
### 위치
- 메인화면 오른쪽 상단 `로그아웃`

### 사용 순서
1. `로그아웃` 클릭
2. 확인 메시지가 나오면 진행
3. 로그인 화면으로 돌아갑니다.

## 4. 메인화면 이해하기

메인화면은 크게 4개 영역으로 봐두면 편합니다.

### 1) 상단 주요 메뉴
- `품목/재고`
- `신규 렌탈 거래처 등록`
- `거래처 설정`
- `납품내역`
- `자료집계`
- `렌탈 관리`
- `환경설정`
- `휴지통`
- `로그아웃`

### 2) 요약 / 알림 영역
- 계약서 알림
- 렌탈 예정 / 지연
- 동기화 상태

### 3) 거래 조회 영역
상단 필터로 아래를 조절합니다.
- `기간`
- `유형`
- `담당지점`
- `조회`
- `필터초기화`
- `즐겨찾기 이동`
- `즐겨찾기 추가/해제`

### 4) 실무 버튼 영역
- `수금 입력`
- `전표 인쇄[F9]`
- `판매작성`
- `구매작성`
- `견적 발주`
- `삭제`
- `계약서 보기`

> 주의  
> `계약서 보기`는 거래처를 선택했고, 그 거래처에 연결된 계약서가 있을 때 사용합니다.

## 5. 공통 사용 방법

## 검색
대부분의 화면 상단에 `검색` 또는 `찾기` 버튼이 있습니다.

예:
- 거래처 관리: `검색` + `찾기`
- 납품내역: `거래처 검색` + `검색`
- 메인화면: 고객 선택과 기간/유형 필터

## 필터
주요 필터 예시:
- 거래처 관리: `고객분류`, `담당지점`, `정렬`
- 품목/재고: `재고방식`
- 렌탈 자산 / 설치현황: `품목분류`, `담당지점`, `상태`
- 렌탈 청구관리: `담당지점`, `담당계정`, `상태`, `알림대상만`, `기준일`

## 저장
화면마다 저장 방식이 다릅니다.

### 명시적으로 `저장` 버튼을 누르는 화면
- 거래처 등록 / 수정
- 품목 / 재고관리
- 렌탈 자산 / 설치현황
- 렌탈 청구관리
- 환경설정 일부 화면

### 문서 입력 흐름에 따라 저장되는 화면
- 판매작성 / 구매작성 전표 창은 별도 큰 `저장` 버튼이 보이지 않을 수 있습니다.
- 전표 창에서는 `항목추가`, `항목수정`, 수금/지급 창 진입 전 저장 확인, 창 닫기 시 자동저장 흐름이 같이 동작합니다.
- 화면 상단에 `이 화면은 자동저장 방식으로 동작합니다` 안내 문구가 표시됩니다.

## 삭제
- 일반 삭제는 대부분 **휴지통 이동** 개념입니다.
- 완전 삭제는 `환경설정 > 휴지통`에서 처리합니다.
- 연결 데이터가 남아 있으면 영구삭제가 막힐 수 있습니다.

## 인쇄
- 메인화면: `전표 인쇄[F9]`
- 전표 창: `인쇄하기[F9]`
- 출력 형식 편집: `출력물 편집`
- 세금계산서 인쇄: `세금계산인쇄`
- 인쇄창 이름: `거래플랜 인쇄`
- 복합기나 프린터가 목록에 잡히지 않으면 `PDF 저장`으로 먼저 저장한 뒤 복합기 연결 PC 또는 다른 PC에서 출력합니다.
- 프린터가 오프라인이면 `속성...`으로 상태를 확인하거나 `PDF 저장` / `파일 저장(XPS)`을 사용합니다.
- 일부 페이지만 출력하려면 `페이지`에 `1,3,5-12`처럼 입력합니다.
- 여러 부를 출력할 때는 `인쇄 매수`와 `한 부씩 인쇄`를 확인합니다.

## 6. 거래처 설정 사용법

## 메뉴 구조
상단 `거래처 설정` 클릭 시 하위 메뉴가 열립니다.

- `거래처 등록`
- `거래처 삭제`
- `거래처 관리`

### 권장 사용법
- 신규 등록: `거래처 설정 > 거래처 등록`
- 목록 조회 / 수정 / 삭제: `거래처 설정 > 거래처 관리`

---

## 6-1. 거래처 등록

### 진입
- `거래처 설정 > 거래처 등록`

### 주요 입력 항목
- `거래처명`
- `고객분류`
- `거래구분`
- `가격등급`
- `담당지점`
- `대표자명`
- `대표전화`
- `사업자번호`
- 주소 / 메모 등

### 상단 버튼
- `저장`
- `연속입력 (F6)`
- `닫기 (F12)`
- `PDF 추가`
- `열기`
- `대표 지정`
- `삭제`

### 사용 순서
1. `거래처 설정 > 거래처 등록` 진입
2. 거래처 기본 정보 입력
3. 필요하면 계약서 PDF를 `PDF 추가`
4. `저장`
5. 계속 입력할 경우 `연속입력 (F6)`

### 계약서 보기까지 이어서 확인하는 방법
1. 거래처 등록 또는 수정 화면에서 계약서 PDF를 `PDF 추가`
2. `저장`
3. 메인화면으로 돌아와 해당 거래처 선택
4. `계약서 보기`
5. 연결된 계약서 PDF가 열리면 정상입니다.

### 주의사항
- `고객분류`와 `거래구분`은 다른 항목입니다.
- 현재 거래구분은 아래 값만 사용합니다.
  - `매출`
  - `매입`
  - `매출/매입`

## 6-2. 거래처 관리

### 진입
- `거래처 설정 > 거래처 관리`

### 상단 기능
- `거래처 신규 등록`
- `선택 거래처 수정`
- `선택 거래처 삭제`
- `담당지점 저장`
- `새로고침`
- `닫기`

### 검색 / 필터
- `검색`
- `찾기`
- `고객분류`
- `담당지점`
- `정렬`

### 목록에서 확인 가능한 정보
- 거래구분
- 거래처명
- 계약구분
- 계약상태
- 계약서
- 고객분류
- 담당지점
- 대표전화
- 만료일
- 사업자번호
- 상태

### 추천 업무 흐름
1. `검색`으로 거래처 찾기
2. `고객분류`, `담당지점`, `정렬`로 범위 좁히기
3. 거래처 선택
4. `선택 거래처 수정` 또는 `선택 거래처 삭제`

### 삭제 시 주의
- 삭제된 거래처는 휴지통으로 이동합니다.
- 전표, 렌탈 청구, 연결 데이터가 남아 있으면 이후 영구삭제가 제한될 수 있습니다.

## 7. 품목 / 재고관리 사용법

### 진입
- 메인 상단 `품목/재고`

### 상단 버튼
- `등록 (신규)`
- `저장`
- `삭제`
- `재고이동`

### 주요 입력 항목
- `품명`
- `품목분류`
- `규격`
- `단위`
- `재고방식`
- `운영유형`
- `매입단가`
- `매출단가`
- `소매단가`
- `A단가`, `B단가`, `C단가`
- `보관위치`
- `기본Box수량`
- `간단메모`

### 조회 / 필터
- 검색어 입력
- `재고방식` 필터
- 지점별 재고 탭/수량 확인

### 신규 품목 등록 순서
1. `등록 (신규)` 클릭
2. 품목 기본정보 입력
3. 가격 / 단위 / 재고 관련 정보 입력
4. `저장`

### 삭제 순서
1. 목록에서 품목 선택
2. `삭제`
3. 필요 시 휴지통에서 복원

### 권한 주의
- 품목 저장 / 삭제는 **관리자 또는 동등 권한 계정**이 필요합니다.
- 일반 사용자는 조회 중심으로 사용하는 경우가 많습니다.

## 8. 판매작성 / 구매작성 사용법

## 진입
메인화면에서 아래 중 하나를 클릭합니다.
- `판매작성`
- `구매작성`
- `견적 발주`

## 전표 창에서 보이는 주요 버튼
- `신규전표(F6)` 또는 `신규구매(F6)` / `신규발주(F6)`
- `수금 입력` 또는 `지급 입력`
- `닫기(F12)`
- `항목추가`
- `항목수정`
- `항목삭제`
- `이전기록불러오기`
- `인쇄하기[F9]`
- `출력물 편집`
- `세금계산인쇄`
- `세금계산서 발행 완료`

## 기본 입력 흐름
1. 전표 창 열기
2. 거래처 선택
3. 날짜 / 담당지점 / 메모 등 확인
4. 품목 검색
5. `항목추가`
6. 필요 시 `항목수정`, `항목삭제`
7. 인쇄 또는 수금/지급으로 후속 처리

## 저장 방식 주의
- 전표 창은 일반 입력 화면처럼 큰 `저장` 버튼이 보이지 않을 수 있습니다.
- 대신 아래 흐름에서 저장이 동작합니다.
  - 전표 내부 저장 명령
  - 수금/지급 창 진입 전 저장 확인
  - 창 닫기 시 자동저장 시도
- 창을 닫을 때 저장되지 않은 변경이 있으면 안내가 나올 수 있습니다.

## 세금계산서 관련
- `세금계산인쇄`: 세금계산서 문서 출력용
- `세금계산서 발행 완료`: 발행 여부 체크용

## 출력물 편집
- `출력물 편집`에서 인쇄용 항목/표현을 조정합니다.
- 이후 `인쇄하기[F9]`로 미리보기/인쇄를 진행합니다.
- `거래플랜 인쇄` 창에서 프린터를 선택하거나 `PDF 저장` / `파일 저장(XPS)`으로 문서를 저장할 수 있습니다.
- 복합기 연결이 불안정하면 바로 인쇄보다 `PDF 저장` 후 출력하는 방법이 안전합니다.

## 9. 수금 입력 / 지급 입력 사용법

## 진입 방법
### 방법 1

## 10-1. 신규 렌탈 거래처 등록

### 진입
- 메인 상단 `신규 렌탈 거래처 등록`
- 또는 `렌탈 청구관리` 내부 `신규 렌탈 거래처 등록`

### 단계
1. `거래처정보`
2. `렌탈 기본정보`
3. `임대료 청구 설정`
4. `장비 연결`
5. `표시품목/내부 장비 구성`
6. `최종 확인`

### 특징
- 작성 중 항목을 추가하고 창을 닫아도 **드래프트 자동저장**이 동작합니다.
- 다시 열면 이전 작성 내역이 복원됩니다.
- 최종 저장 성공 시 드래프트는 정리됩니다.
- `표시품목`은 거래명세서/청구서에 인쇄될 출력 라인입니다.
- `거래처 임대 자산`은 실제 청구서 만들기와 전표 생성 대상이 되는 자산입니다.
- 표시 품목명만 바꿔도 자산이 자동으로 추가/삭제되지는 않습니다. 실제 청구 대상은 거래처 임대 자산 목록에서 확인합니다.

### 주요 필터

- `담당계정`
- `상태`

### 목록에서 보는 항목
- `거래처명`

- `상태`

- `미수금`
- `이상`
- `청구처`
- `설치처`
- `담당지점`

### 임대료 청구 기본 흐름
1. 신규 거래처면 `신규 렌탈 거래처 등록`
2. 기존 거래처면 `신규 설정` 또는 목록 선택
3. 청구 정보 입력 후 `저장`
4. 청구 대상 거래처 선택
5. `선택 월 청구서 만들기`
6. 필요 시 `입금 등록`
7. 완료 시 `완납 처리`

### 표시품목 / 거래처 임대 자산 구분
- `청구서 표시 품목`은 거래명세서와 청구서에 보여줄 **표시 라인**입니다.
- `거래처 임대 자산`은 실제 청구/전표 생성에 포함되는 **렌탈 자산 목록**입니다.
- 표시 품목에서 품명, 규격, 라인유형을 정하고, 실제 청구 대상 장비는 `거래처 임대 자산 연결`로 연결합니다.
- 표시 품목명을 수정해도 거래처 임대 자산이 자동으로 바뀌지 않습니다.
- 거래처 임대 자산에서 장비를 삭제한 뒤 저장하면 해당 장비만 현재 청구 연결에서 해제됩니다.
- 라인유형이 `개별`인 표시품목은 같은 모델명끼리 청구서 만들기 시 수량이 합산되어 전표 라인에 반영됩니다.
- 청구서 표시 품목이 비어 있거나 거래처 임대 자산이 없으면 실제 청구/전표 생성 대상이 누락될 수 있으므로 저장 전 반드시 확인합니다.

### 자동저장 주의
- 청구 편집 중 작성 내용은 드래프트로 자동저장됩니다.
- 저장 전에 창을 닫아도 다음에 다시 열면 이어서 작성할 수 있습니다.

## 일반유저(User)
일반유저 계정은 보통 아래 제약이 있습니다.
- 지점 또는 자신에게 허용된 범위 중심 조회
- 품목 저장 / 삭제 제한 가능
- 사용자 관리 / 선택값 관리 / 일부 환경설정 제한
- 렌탈/납품/금액 조회 범위가 좁을 수 있음

### 화면에서 보일 수 있는 권한 안내 예시
- `현재 계정은 품목을 저장할 권한이 없습니다. 관리자 계정으로 로그인하거나 관리자에게 저장을 요청하세요.`
- `회사 정보는 관리자 권한이 있는 계정만 수정할 수 있습니다.`
- `업체 / 데이터 권한 화면은 관리자 계정만 조회하거나 변경할 수 있습니다.`

> 실제 점검 기준  
> `usenet` 계정은 관리자(Admin/USENET, TenantAll) 권한으로 확인했고,  
> `yeonsu` 계정은 일반유저(User/YEONSU, OfficeOnly) 권한으로 확인했습니다.

## 15. 자주 헷갈리는 부분

### 1) 로그인 직후 알림 창
- 로그인 후 `대시보드 알림` 창이 먼저 뜰 수 있습니다.
- 팝업 문구를 확인한 뒤 `확인`을 누르면 메인화면으로 이동해 계속 작업할 수 있습니다.

### 2) 판매 / 구매 전표 창의 저장 방식
- 일반 등록 화면처럼 큰 저장 버튼이 보이지 않을 수 있습니다.
- 항목 편집, 수금/지급 진입 전 저장 확인, 닫기 시 자동저장 흐름을 함께 사용합니다.

### 3) 계약서 보기 버튼이 안 눌리는 경우
- 거래처를 먼저 선택해야 합니다.
- 해당 거래처에 연결된 계약서가 있어야 합니다.

### 4) 휴지통에서 영구삭제가 안 되는 경우
- 연결 전표, 거래내역, 수금/지급, 렌탈 데이터가 남아 있으면 영구삭제가 제한됩니다.
- 이 경우 `삭제 차단 사유`를 먼저 확인하세요.

### 5) 담당지점 관리 탭
- 현재 구조상 canonical 3개 담당지점(USENET, ITWORLD, YEONSU) 기준으로 유지됩니다.
- 편집 가능한 지점 마스터 관리 화면으로 이해하지 마세요.

## 16. 문제 발생 시 먼저 확인할 항목

### 로그인 관련
- 아이디 / 비밀번호
- 서버 연결 가능 여부
- 저장된 계정 정보가 오래되지 않았는지

### 저장 / 조회 관련
- 필수 입력값 누락 여부
- 저장 후 목록 `새로고침`
- 권한 부족 여부

### 동기화 관련
- `환경설정 > 동기화`에서 상태 확인
- `동기화 진단` 확인
- 서버 반영 대기 메시지 확인

### 휴지통 관련
- 연결 데이터 때문에 영구삭제가 막힌 것은 아닌지
- 중복 거래처 정리는 `연결 이동 후 영구삭제`가 더 적절한지

### 인쇄 관련
- 먼저 `인쇄하기[F9]` 또는 `전표 인쇄[F9]`로 미리보기 확인
- `거래플랜 인쇄` 창에서 프린터가 보이는지 확인
- 프린터가 없거나 복합기 연결이 안 되면 `PDF 저장`으로 저장 후 다른 PC/복합기에서 출력
- 프린터가 보이지만 출력이 안 되면 `속성...`과 Windows 프린터 설정을 같이 점검
- 페이지 일부만 출력할 때는 페이지 범위 형식이 `1,3,5-12`처럼 맞는지 확인

### 업데이트 관련
- `버전 / 업데이트` 또는 업데이트 알림 확인
- 사용자 PC가 최신 설치본인지 확인

## 마지막 정리

처음 사용하는 경우 권장 순서는 아래와 같습니다.

1. 로그인
2. 메인화면 구조 익히기
3. `거래처 설정 > 거래처 등록`
4. `품목/재고`에서 품목 등록
5. 메인에서 `판매작성` 또는 `구매작성`
6. `수금 입력` / `지급 입력`
7. 렌탈이 있으면 `신규 렌탈 거래처 등록` 또는 `렌탈 관리`
8. 필요 시 `전표 인쇄[F9]`, `자료집계`, `환경설정 > 동기화 / 백업`

> 권장  
> 처음에는 관리자 계정으로 전체 흐름을 익힌 뒤, 일반유저 계정에서 보이는 화면 차이를 확인하는 방식이 가장 이해하기 쉽습니다.
