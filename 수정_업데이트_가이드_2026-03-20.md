# 거래플랜 수정 / 업데이트 가이드

## 1. 가장 빠른 통합 릴리스 방법
PC 설치파일, 모바일 APK, 업데이트 자산, NAS 배포까지 한 번에 처리하려면 아래 명령을 사용합니다.

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\release\Publish-GeoraePlanFullRelease.ps1" -ProjectRoot "D:\거래플랜" -SigningConfigPath "D:\거래플랜\Mobile\GeoraePlan.Mobile.App\android-signing.local.json" -DeployToNas
```

생성/반영 결과:
- PC 설치파일: `D:\거래플랜\배포\거래플랜-PC-설치패키지.exe`
- 모바일 APK: `D:\거래플랜\배포\거래플랜-안드로이드-v0.2.4-signed.apk`
- 업데이트 매니페스트: `D:\거래플랜\배포\업데이트\manifest\stable.json`
- NAS live 반영: `\\192.0.2.10\docker\georaeplan\app\live`

## 2. 개별 작업만 할 때
### 2-1. 솔루션 빌드
```powershell
cd "D:\거래플랜"
dotnet build "거래플랜.sln" -c Release
```

### 2-2. PC 설치파일 다시 만들기
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\release\Build-GeoraePlanDesktopInstaller.ps1" -ProjectRoot "D:\거래플랜"
```

### 2-3. 안드로이드 APK 다시 만들기
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\mobile\Build-GeoraePlanAndroidApk.ps1" -ProjectRoot "D:\거래플랜" -SigningConfigPath "D:\거래플랜\Mobile\GeoraePlan.Mobile.App\android-signing.local.json"
```

> 참고: Release 빌드는 기본적으로 `PublishTrimmed=false`로 동작하도록 설정해 두었습니다.

### 2-4. 내부 업데이트 자산 다시 만들기
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\release\Publish-GeoraePlanUpdateAssets.ps1" -ProjectRoot "D:\거래플랜"
```

### 2-5. NAS 실서버 반영
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\nas\Publish-GeoraeplanNasRelease.ps1" -ProjectRoot "D:\거래플랜" -MirrorToLive
```

## 3. 버전 올리는 위치
### PC
파일:
- `D:\거래플랜\Desktop\거래플랜.Desktop.App\거래플랜.Desktop.App.csproj`

수정 항목:
- `<Version>`
- `<FileVersion>`

### 모바일
파일:
- `D:\거래플랜\Mobile\GeoraePlan.Mobile.App\GeoraePlan.Mobile.App.csproj`

수정 항목:
- `<ApplicationDisplayVersion>`
- `<ApplicationVersion>`

## 4. 권장 운영 흐름
1. 소스 수정
2. 버전 올리기
3. `Publish-GeoraePlanFullRelease.ps1` 실행
4. PC/모바일에서 업데이트 확인
5. NAS health 확인
