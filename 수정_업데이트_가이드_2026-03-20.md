# 거래플랜 수정 / 빌드 / 업데이트 가이드 (Linux PC 기준)

## 1. 가장 권장하는 전체 릴리스 명령
아래 명령 1회로
- PC 설치파일 생성
- Android APK 생성
- 업데이트 자산 생성
- Linux PC live 반영
까지 순서대로 실행합니다.

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\release\Publish-GeoraePlanFullRelease.ps1" `
  -ProjectRoot "D:\거래플랜" `
  -SigningConfigPath "D:\거래플랜\Mobile\GeoraePlan.Mobile.App\android-signing.local.json" `
  -DeployToLinuxPc `
  -FailOnOperationalWarnings
```

> 현재 운영 서버는 Linux PC `/srv/georaeplan`입니다.

---

## 2. Linux PC 자동 반영 방식
배포 스크립트는 release를 Linux PC에 SSH로 업로드한 뒤 `/srv/georaeplan/ops/apply-release.sh <release-id>`를 실행합니다.

운영 기본값:

```dotenv
LINUX_SSH_HOST=192.168.0.199
LINUX_SSH_PORT=2222
LINUX_SSH_USER=itw
LINUX_SSH_KEY_PATH=C:\Users\beene\.ssh\itwserver_codex_ed25519
LINUX_REMOTE_OPS_PATH=/srv/georaeplan/ops
PUBLIC_BASE_URL=https://trade.2884.kr
```

서버 publish/live 반영만 다시 할 때:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\linux\Publish-GeoraeplanLinuxPcRelease.ps1" `
  -ProjectRoot "D:\거래플랜" `
  -MirrorToLive `
  -FailOnOperationalWarnings
```

Linux PC 전용 래퍼는 다음을 수행합니다.
- `https://trade.2884.kr/healthz`, `https://work.2884.kr/healthz`, `https://itw.2884.kr/` 사전/사후 확인
- SSH로 `/srv/georaeplan/ops/apply-release.sh` 존재 및 `bash -n` 확인
- legacy config sync를 기본 차단해 Linux PC `/srv/georaeplan/ops` 덮어쓰기 방지
- 기존 release apply 로직을 재사용하되 Linux SSH 설정을 강제 전달

## 3. 개별 작업 명령

### 3-1. 솔루션 빌드
```powershell
cd "D:\거래플랜"
dotnet build "D:\거래플랜\거래플랜.sln" -c Release
```

### 3-2. PC 설치파일 생성
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\release\Build-GeoraePlanDesktopInstaller.ps1" -ProjectRoot "D:\거래플랜"
```

### 3-3. Android APK 생성
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\mobile\Build-GeoraePlanAndroidApk.ps1" -ProjectRoot "D:\거래플랜" -SigningConfigPath "D:\거래플랜\Mobile\GeoraePlan.Mobile.App\android-signing.local.json"
```

### 3-4. 업데이트 자산 생성
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\release\Publish-GeoraePlanUpdateAssets.ps1" -ProjectRoot "D:\거래플랜"
```

### 3-5. Linux PC live 반영만 다시 하기
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\linux\Publish-GeoraeplanLinuxPcRelease.ps1" -ProjectRoot "D:\거래플랜" -MirrorToLive -FailOnOperationalWarnings
```

> 유료 납품/엄격 release에서는 live 전/후 operational gate를 생략하지 않고 `-FailOnOperationalWarnings`를 사용합니다.
