# 거래플랜 수정 / 업데이트 가이드

## 1. 가장 빠른 통합 릴리스 방법
PC 설치파일, 모바일 APK, 업데이트 자산 생성, NAS 반영까지 한 번에 처리하려면 아래 명령을 사용합니다.

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\release\Publish-GeoraePlanFullRelease.ps1" -ProjectRoot "D:\거래플랜" -SigningConfigPath "D:\거래플랜\Mobile\GeoraePlan.Mobile.App\android-signing.local.json" -DeployToNas
```

이 명령이 수행하는 작업:
1. `dotnet build` Release
2. PC 설치파일 생성
3. Android 서명 APK 생성
4. 업데이트 매니페스트/다운로드 자산 생성
5. NAS `releases/<release-id>` 복사
6. NAS `apply-release.sh` 실행
7. NAS 컨테이너 재기동 및 healthz 확인

## 2. NAS 자동 재기동이 되도록 사전 설정하는 방법
배포 PC에서 NAS의 `apply-release.sh`를 호출할 수 있도록 SSH 정보가 필요합니다.

설정 위치:
- 운영 NAS 공유 폴더: `\\192.0.2.10\docker\georaeplan\ops\.env`
- 예시 파일: `D:\거래플랜\infra\nas\.env.api.example.invalid.example`

필수/권장 항목:
```dotenv
NAS_SSH_HOST=192.0.2.10
NAS_SSH_PORT=22
NAS_SSH_USER=
NAS_SSH_KEY_PATH=
NAS_REMOTE_OPS_PATH=/volume1/docker/georaeplan/ops
```

주의:
- `NAS_SSH_USER`는 필수입니다.
- `NAS_SSH_KEY_PATH`는 **Windows 배포 PC 기준 경로**입니다.
  - 예: `C:\Users\beene\.ssh\id_ed25519`
- 키 기반 로그인이 가능하면 가장 좋고, 키가 없으면 수동 실행 시 SSH 비밀번호 입력을 요구할 수 있습니다.

## 3. 개별 작업만 할 때
### 3-1. 솔루션 빌드
```powershell
cd "D:\거래플랜"
dotnet build "거래플랜.sln" -c Release
```

### 3-2. PC 설치파일 다시 만들기
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\release\Build-GeoraePlanDesktopInstaller.ps1" -ProjectRoot "D:\거래플랜"
```

### 3-3. Android APK 다시 만들기
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\mobile\Build-GeoraePlanAndroidApk.ps1" -ProjectRoot "D:\거래플랜" -SigningConfigPath "D:\거래플랜\Mobile\GeoraePlan.Mobile.App\android-signing.local.json"
```

### 3-4. 업데이트 자산만 다시 만들기
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\release\Publish-GeoraePlanUpdateAssets.ps1" -ProjectRoot "D:\거래플랜"
```

### 3-5. NAS 반영만 다시 하기
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\nas\Publish-GeoraeplanNasRelease.ps1" -ProjectRoot "D:\거래플랜" -MirrorToLive
```

## 4. 예전 방식 직접 미러링이 꼭 필요한 경우
권장 방식은 아니지만, SSH 자동 적용 구성이 아직 없을 때만 아래처럼 강제로 예전 방식을 허용할 수 있습니다.

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\nas\Publish-GeoraeplanNasRelease.ps1" -ProjectRoot "D:\거래플랜" -MirrorToLive -AllowLegacyLiveMirror
```

주의:
- 이 방식은 `app\live`를 직접 덮어쓰는 방식이라, 실행 중 파일 교체 상황에서 오류가 날 수 있습니다.
- 가능하면 반드시 `apply-release.sh` 자동 실행 방식을 사용하세요.

## 5. 배포본 위치
- PC 설치파일(실사용): `D:\거래플랜\배포\거래플랜-PC-설치패키지.exe`
- Android APK(실사용): `D:\거래플랜\배포\거래플랜-안드로이드-v0.2.4-signed.apk`
- 관리자용 MSI/ZIP: `D:\거래플랜\배포\관리자용`
- 업데이트 매니페스트: `D:\거래플랜\배포\업데이트\manifest\stable.json`

## 6. 배포 후 확인
### 로컬/NAS 외부 확인
```powershell
Invoke-WebRequest -UseBasicParsing "https://api.example.invalid/healthz"
Invoke-WebRequest -UseBasicParsing "https://api.example.invalid/updates/manifest?channel=stable"
```

### 업데이트 다운로드 확인
```powershell
Invoke-WebRequest -UseBasicParsing "https://api.example.invalid/updates/download/android/tradeplan-android-v0.2.4.apk" -OutFile "$env:TEMP\tradeplan-android-v0.2.4.apk"
```

정상이면:
- `healthz`는 200
- `manifest`는 200
- APK 다운로드도 200으로 떨어져야 합니다.
