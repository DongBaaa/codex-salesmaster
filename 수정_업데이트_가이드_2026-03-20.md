# 거래플랜 수정 / 업데이트 가이드

## 1. 소스를 수정할 때
1. 소스 수정
2. 빌드 확인
```powershell
cd "D:\거래플랜"
dotnet build "거래플랜.sln" -c Release
```
3. PC 로컬 확인
```powershell
cmd /c "D:\거래플랜\배포\전체실행.cmd"
```
4. 필요 시 모바일 APK 다시 생성
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\mobile\Build-GeoraePlanAndroidApk.ps1" -ProjectRoot "D:\거래플랜" -SigningConfigPath "D:\거래플랜\Mobile\GeoraePlan.Mobile.App\android-signing.local.json"
```

## 2. PC 설치파일 다시 만들기
일반 배포용 EXE + 관리자용 MSI/ZIP을 같이 다시 만듭니다.
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\release\Build-GeoraePlanDesktopInstaller.ps1"
```
생성 결과:
- 일반 사용자용: `D:\거래플랜\배포\설치패키지\거래플랜-PC-설치패키지.exe`
- 관리자용 보관: `D:\거래플랜\배포\설치패키지\관리자용`

## 3. 안드로이드 APK / AAB 다시 만들기
APK:
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\mobile\Build-GeoraePlanAndroidApk.ps1" -ProjectRoot "D:\거래플랜" -SigningConfigPath "D:\거래플랜\Mobile\GeoraePlan.Mobile.App\android-signing.local.json"
```
AAB:
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\mobile\Build-GeoraePlanAndroidBundle.ps1" -ProjectRoot "D:\거래플랜" -SigningConfigPath "D:\거래플랜\Mobile\GeoraePlan.Mobile.App\android-signing.local.json"
```

## 4. 내부 업데이트용 배포 자산 만들기
PC 내부 업데이트는 ZIP을 사용하므로 관리자용 ZIP이 필요합니다.
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\release\Publish-GeoraePlanUpdateAssets.ps1" -ProjectRoot "D:\거래플랜"
```
생성 위치:
- `D:\거래플랜\배포\업데이트\manifest`
- `D:\거래플랜\배포\업데이트\downloads`

## 5. NAS 실서버로 반영하기
서버 코드나 업데이트 자산을 NAS에 반영할 때:
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\nas\Publish-GeoraeplanNasRelease.ps1" -ProjectRoot "D:\거래플랜" -MirrorToLive
```

## 6. 버전을 올릴 때
### PC 버전
- 파일: `D:\거래플랜\Desktop\거래플랜.Desktop.App\거래플랜.Desktop.App.csproj`
- 항목:
  - `<Version>`
  - `<FileVersion>`

### Android 버전
- 파일: `D:\거래플랜\Mobile\GeoraePlan.Mobile.App\GeoraePlan.Mobile.App.csproj`
- 항목:
  - `<ApplicationDisplayVersion>`
  - `<ApplicationVersion>`

버전을 올린 뒤 다시 빌드/설치파일 생성/업데이트 자산 생성을 해야 합니다.

## 7. 실제 사용 흐름 요약
- 사용자 배포: EXE 하나 전달
- 관리자 보관: MSI/ZIP 보관
- 앱 내부 업데이트: ZIP + APK를 NAS 업데이트 자산으로 배포
- 관리자/일반 권한 구분: 설치파일이 아니라 로그인 권한으로 처리
