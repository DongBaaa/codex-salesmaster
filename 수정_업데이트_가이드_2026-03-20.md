## 2. NAS 자동 반영 방식
배포 스크립트는 release를 NAS에 복사한 뒤 아래 2가지 방식 중 하나로 `apply-release.sh`를 실행합니다.

### 2-1. 우선순위 1: SSH 직접 실행
운영 NAS의 `ops/.env` 또는 배포 PC 환경변수에 아래를 넣으면,
배포 PC가 NAS에 SSH 접속해 `apply-release.sh <release-id>`를 바로 실행합니다.

```dotenv
NAS_SSH_HOST=192.0.2.10
NAS_SSH_PORT=22
NAS_SSH_USER=boss
NAS_SSH_KEY_PATH=C:\Users\beene\.ssh\tradeplan_nas_ed25519
NAS_REMOTE_OPS_PATH=/volume1/docker/georaeplan/ops
```

### 2-2. 우선순위 2: NAS 작업 스케줄러 자동 적용
SSH를 못 쓰는 경우에는 NAS 작업 스케줄러가 `auto-apply-release.sh`를 주기적으로 실행하도록 구성합니다.

운영 NAS `ops/.env`:

```dotenv
NAS_SCHEDULED_APPLY_ENABLED=true
```

NAS 작업 스케줄러의 사용자 정의 스크립트 예시:

```bash
cd /volume1/docker/georaeplan/ops && chmod +x ./auto-apply-release.sh ./apply-release.sh ./health-check.sh ./_common.sh && /bin/bash ./auto-apply-release.sh
```

권장 스케줄:
- 1분마다
- 사용자: `boss` 또는 Docker 관리 권한이 있는 관리자 계정

이 방식에서는 배포 스크립트가 `ops/state/pending-release.txt`에 release-id를 기록하고,
NAS가 해당 파일을 감지해 `apply-release.sh`를 로컬에서 실행합니다.

### 2-3. 임시 예외: 직접 live 미러링
정말 필요한 경우에만 아래 옵션으로 예전 방식을 강제로 허용할 수 있습니다.

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\거래플랜\tools\nas\Publish-GeoraeplanNasRelease.ps1" -ProjectRoot "D:\거래플랜" -MirrorToLive -AllowLegacyLiveMirror
```

이 방식은 실행 중 파일을 직접 덮어쓰므로 권장하지 않습니다.
