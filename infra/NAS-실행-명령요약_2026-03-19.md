# 거래플랜 NAS 실행 명령 요약

- 기준일: 2026-03-19
- 목적: 거래플랜 API를 NAS에 올리고 `/healthz` 와 PC/Android 연결까지 빠르게 점검하는 1장 요약

## 1. 로컬 PC에서 ops 파일 복사
```powershell
cd "D:\거래플랜"
powershell -ExecutionPolicy Bypass -File .\tools\nas\Sync-GeoraeplanNasConfig.ps1
```

## 2. NAS `ops\.env` 만들기
NAS에 아래 파일을 복사해 `.env` 로 저장:

```text
\\192.0.2.10\docker\georaeplan\ops\.env.public.example
```

운영 전 반드시 바꿀 값:
- `POSTGRES_PASSWORD`
- `JWT_SIGNING_KEY`
- `SECURITY_REQUIRE_HTTPS_FORWARDED_PROTO=true`
- `SEED_USERS_ADMIN_PASSWORD`
- `PUBLIC_BASE_URL=https://api.example.invalid`

## 3. 로컬 PC에서 서버 publish + live 반영
```powershell
cd "D:\거래플랜"
powershell -ExecutionPolicy Bypass -File .\tools\nas\Publish-GeoraeplanNasRelease.ps1 -MirrorToLive
```

## 4. NAS에서 컨테이너 실행
```bash
cd /volume1/docker/georaeplan/ops
docker compose --env-file .env up -d
```

## 5. NAS 내부 상태 확인
```bash
docker compose ps
docker compose logs -f api
curl http://127.0.0.1:18082/healthz
```

정상 응답 예:
```json
{"status":"ok","utc":"..."}
```

## 6. 외부 도메인 상태 확인
현재 2026-03-19 기준 이 PC에서는 `api.example.invalid` DNS 해석이 실패했다. 배포 후 아래가 성공해야 한다.

```powershell
curl https://api.example.invalid/healthz
```

성공 기준:
- Synology 404 가 아님
- API JSON 이 옴
- 인증서 경고가 없음

## 7. PC 배포본 NAS 주소로 변경
```powershell
cd "D:\거래플랜\배포"
powershell -ExecutionPolicy Bypass -File .\Set-ApiBaseUrl.ps1 -BaseUrl https://api.example.invalid
```

## 8. Android 앱 서버 주소 입력
앱 설정에서 서버 주소를 아래로 저장:

```text
https://api.example.invalid
```
