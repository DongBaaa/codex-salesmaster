# 거래플랜 NAS 보안 체크리스트

- 기준일: 2026-03-19
- 목적: 거래플랜을 NAS 실운영할 때 최소한의 보안 하드닝 항목을 빠르게 점검하기 위한 문서

## 1. 네트워크 노출
- [ ] PostgreSQL 포트는 `127.0.0.1:15434` 로만 바인딩되어 있는가
- [ ] API 포트는 `127.0.0.1:18082` 로만 바인딩되어 있는가
- [ ] 외부 공개는 Synology reverse proxy 또는 Tailscale 로만 하는가
- [ ] NAS 방화벽에서 불필요한 포트가 열려 있지 않은가

## 2. HTTPS
- [ ] `api.example.invalid` 에 유효한 HTTPS 인증서가 적용되어 있는가
- [ ] reverse proxy 대상이 `http://127.0.0.1:18082` 인가
- [ ] `SECURITY_REQUIRE_HTTPS_FORWARDED_PROTO=true` 인가
- [ ] `https://api.example.invalid/healthz` 가 200 OK 인가

## 3. 인증/비밀번호
- [ ] `JWT_SIGNING_KEY` 가 32자 이상 랜덤 문자열인가
- [ ] 기본 DB 비밀번호 `CHANGE_THIS_LOCAL_POSTGRES_PASSWORD` 를 더 이상 쓰지 않는가
- [ ] `SEED_USERS_ADMIN_PASSWORD` 를 강한 값으로 설정했는가
- [ ] `SEED_USERS_USER_PASSWORD`, `SEED_USERS_ITW_PASSWORD` 를 실제 값으로 넣었거나 빈값/`__DISABLE__` 로 비활성화했는가
- [ ] 운영 시작 후 기본 seed 비밀번호를 그대로 쓰지 않는가

## 4. API 보호
- [ ] Rate limiting 이 켜져 있는가 (`SECURITY_ENABLE_RATE_LIMITING=true`)
- [ ] 로그인 제한이 분당 10회 수준으로 설정되어 있는가
- [ ] CORS 가 `AllowAnyOrigin` 이 아닌 제한 모드인가
- [ ] 필요 시 `SECURITY_ALLOWED_CORS_ORIGIN_0` 만 허용되어 있는가
- [ ] 보안 헤더(`nosniff`, `DENY`, `HSTS`)가 응답에 붙는가

## 5. 데이터 보호
- [ ] `Database__EnableSqliteFallback=false` 로 NAS 운영 중인가
- [ ] PostgreSQL 데이터 경로가 백업 대상에 포함되어 있는가
- [ ] 계약서 PDF 와 휴지통 데이터가 포함된 서버 DB/파일 백업 절차가 있는가
- [ ] NAS 스냅샷 또는 외부 백업이 정기적으로 수행되는가

## 6. 점검 명령
### 헤더/헬스체크
```powershell
curl -I https://api.example.invalid/healthz
curl https://api.example.invalid/healthz
```

### 로그인 rate limit 점검 예시
```powershell
1..12 | ForEach-Object {
  try {
    Invoke-WebRequest -Uri 'https://api.example.invalid/auth/login' -Method Post -ContentType 'application/json' -Body '{"username":"admin","password":"wrong"}' -UseBasicParsing | Out-Null
    "$_ : unexpected success"
  }
  catch {
    "$_ : $($_.Exception.Response.StatusCode.value__)"
  }
}
```

성공 기준:
- 초반에는 401/400
- 제한 도달 후 429 응답
