## 6. 문제는 아니지만 메뉴얼에 반드시 안내해야 하는 항목

### 1) 로그인 직후 `대시보드 알림` 창이 먼저 뜰 수 있음
- 초보 사용자는 메인화면이 멈춘 것으로 오해할 수 있음
- 메뉴얼에 먼저 `확인`을 누르면 메인화면으로 이동한다고 안내해야 함

### 2) 판매작성 / 구매작성 창은 일반적인 큰 `저장` 버튼 방식이 아님
- 사용자는 저장이 안 된다고 오해할 수 있음
- 실제로는 항목추가/수금진입/닫기 흐름 안에서 저장 동작이 섞여 있음
- 화면 상단 자동저장 안내와 메뉴얼 설명을 같이 유지해야 함

### 3) 휴지통 삭제는 연결 데이터가 있으면 막힐 수 있음
- 과거에는 이 부분이 실제 문제였고, 현재 구조상도 사용자가 헷갈릴 수 있음
- 메뉴얼에서 “영구삭제가 안 되면 연결 전표/거래내역/렌탈 데이터가 남아 있는지 확인” 문구 필요

### 4) 렌탈 등록/청구 화면은 드래프트 자동저장 구조 사용
- 최종 저장 전까지는 임시작성 내용이 남을 수 있음
- 메뉴얼에서 자동저장/최종저장 차이를 설명해야 안전함

6. `담당지점 관리` 탭은 자유 추가/수정 화면으로 이해하면 안 됨
