# 업체 데이터 연동 DB 구조안 (2026-03-20)

## 목표
- 유즈넷/연수구는 같은 업체 그룹 DB를 사용
- 연수구는 자기 지점 데이터만 조회/입력
- 유즈넷은 연수구 포함 상위 조회 가능
- 아이티월드는 완전히 별도 DB 사용
- 업체별로 데이터 연동 허용/차단 정책을 둘 수 있게 함

---

## 권장 아키텍처

### 1. 인증/권한 DB (중앙)
용도:
- 로그인
- 사용자 권한
- 업체/지점 구조
- DB 연결정보
- 연동 정책

### 2. 업무 DB
- `USENET_GROUP` 업무 DB
  - 유즈넷 + 연수구
- `ITWORLD` 업무 DB
  - 아이티월드 전용

즉, 아이티월드는 현재 요구대로 **다른 데이터베이스**로 분리한다.

---

## 테이블 제안

## A. Tenant (업체)
용도:
- 업체 그룹 정의

컬럼 예시:
- `Id` (Guid)
- `TenantCode` (nvarchar 50, unique)
- `TenantName` (nvarchar 100)
- `StorageMode` (nvarchar 30)
  - `SharedDb`
  - `DedicatedDb`
- `DatabaseKey` (nvarchar 100)
- `ParentTenantCode` (nullable)
- `IsActive` (bit)
- `CreatedAtUtc`
- `UpdatedAtUtc`

예시 데이터:
- `USENET_GROUP | 유즈넷 그룹 | SharedDb | TenantDb_UsenetGroup`
- `ITWORLD | 아이티월드 | DedicatedDb | TenantDb_Itworld`

---

## B. TenantOffice (지점)
용도:
- 업체 안의 지점 정의

컬럼 예시:
- `Id`
- `TenantCode`
- `OfficeCode`
- `OfficeName`
- `ParentOfficeCode` (nullable)
- `DefaultScopeType`
  - `OfficeOnly`
  - `TenantAll`
  - `ParentReadable`
- `DefaultWarehouseCode`
- `IsActive`
- `CreatedAtUtc`
- `UpdatedAtUtc`

예시:
- `USENET_GROUP | USENET | 유즈넷 | null | TenantAll`
- `USENET_GROUP | YEONSU | 연수구 | USENET | OfficeOnly`
- `ITWORLD | ITWORLD | 아이티월드 | null | TenantAll`

---

## C. TenantDatabaseBinding
용도:
- 어느 업체가 어느 연결문자열을 쓰는지 지정

컬럼 예시:
- `TenantCode`
- `ConnectionKey`
- `DatabaseProvider`
- `HostSummary`
- `IsPrimary`
- `IsActive`

예시:
- `USENET_GROUP | TenantDb_UsenetGroup | PostgreSql`
- `ITWORLD | TenantDb_Itworld | PostgreSql`

---

## D. DataSharingPolicy
용도:
- 업체/지점 간 데이터 공유 정책

컬럼 예시:
- `Id`
- `SourceTenantCode`
- `SourceOfficeCode` (nullable)
- `TargetTenantCode`
- `TargetOfficeCode` (nullable)
- `ShareCustomers` (bit)
- `ShareInvoices` (bit)
- `SharePayments` (bit)
- `ShareContracts` (bit)
- `ShareReports` (bit)
- `ShareDelivery` (bit)
- `ShareRental` (bit)
- `AllowRead` (bit)
- `AllowWriteBack` (bit)
- `IsEnabled` (bit)
- `CreatedAtUtc`
- `UpdatedAtUtc`

추천 기본값:
- `YEONSU -> USENET`
  - 거래처/전표/수금/계약서/집계 = true
  - `AllowRead = true`
  - `AllowWriteBack = false`
- `ITWORLD -> USENET_GROUP`
  - 전부 false

현재 상태:

  - `TenantCode=ITWORLD`
  - `OfficeCode=ITWORLD`
  - `ScopeType=TenantAll`

## F. 업무 테이블 공통 확장
현재 엔티티 일부는 `OfficeCode`만 있음.
다음 항목을 공통으로 추가 권장:
- `TenantCode`
- `OwnerOfficeCode`
- `CreatedByUserId`
- `UpdatedByUserId`

대상:
- `CustomerMaster`
- `Customer`
- `CustomerContract`
- `Item`
- `Invoice`
- `Payment`
- 납품내역 관련 테이블
- 렌탈 관련 테이블
- 집계용 원장/거래내역 테이블

---

## 업무 데이터 저장 규칙

### 유즈넷/연수구 그룹 DB
- `TenantCode = USENET_GROUP`
- `OwnerOfficeCode = USENET or YEONSU`
- 연수구 사용자가 만든 거래처/전표는 `OwnerOfficeCode = YEONSU`
- 유즈넷 상급권한은 같은 Tenant이므로 조회 가능

### 아이티월드 DB
- `TenantCode = ITWORLD`
- 모든 업무 데이터는 ITWORLD DB에만 저장
- 유즈넷/연수구 DB와 분리

---

## 조회 규칙

### 연수구 사용자
```sql
TenantCode = 'USENET_GROUP'
AND OwnerOfficeCode = 'YEONSU'
```

### 유즈넷 상급권한
```sql
TenantCode = 'USENET_GROUP'
AND OwnerOfficeCode IN ('USENET', 'YEONSU')
```

### 아이티월드 사용자
- 아예 `ITWORLD` DB에 연결
- 추가 office 필터는 선택 사항

---

## 연동 방식

## 1단계: 같은 Tenant 안 연동
연수구가 거래처를 추가/수정
→ 같은 `USENET_GROUP` DB에 저장
→ 유즈넷은 상위 scope로 바로 조회 가능

이것이 가장 중요하고 우선 구현해야 하는 요구사항이다.

## 2단계: 다른 Tenant 간 선택 연동
필요 시 `DataSharingPolicy`로만 허용
예:
- 어떤 공용 마스터만 복제
- 계약서/집계만 공유

단, 아이티월드는 현재 요구상 기본은 완전 분리

---

## 현재 구조와 연결 포인트
현재 파일:
- `D:\거래플랜\Server\거래플랜.Server.Api\Domain\Entities.cs`
- `D:\거래플랜\Server\거래플랜.Server.Api\Services\OfficeScopeService.cs`
- `D:\거래플랜\Shared\거래플랜.Shared.Contracts\Contracts.cs`

현재는 `OfficeCode` 중심이라서:
- 지점 제한은 일부 가능
- 하지만 `TenantCode`, `ScopeType`, `DB 라우팅`, `연동 정책`이 없음

따라서 DB 구조 확장이 필요함.

---

## 추천 마이그레이션 방향
1. 중앙 인증/권한 DB에 `Tenant`, `TenantOffice`, `DataSharingPolicy` 추가
2. `UserAccount`에 `TenantCode`, `ScopeType` 추가
3. 유즈넷/연수구 업무 DB는 기존 DB를 승격 사용
4. 아이티월드는 별도 DB 신규 생성 또는 기존 데이터 분리 이관
5. 업무 엔티티에 `TenantCode`, `OwnerOfficeCode`, `CreatedByUserId`, `UpdatedByUserId` 추가

---

## 핵심 결론
- 연수구 -> 유즈넷 연동은 **같은 Tenant DB + 상위 조회권한**으로 해결
- 아이티월드 분리는 **별도 DB 라우팅**으로 해결
- 업체별 연동 on/off는 `DataSharingPolicy`로 해결
