# 업체 데이터 연동 실제 구현 순서 (2026-03-20)

## 구현 목표
- 연수구가 거래처를 추가/수정하면 유즈넷에서도 조회 가능
- 연수구는 자기 지점 거래처/거래내역만 조회
- 유즈넷은 상급권한으로 연수구 포함 조회
- 아이티월드는 별도 DB 사용
- 업체별로 연동 허용/차단을 환경설정에서 관리

---

## 0단계: 이번 요구사항의 기준 정책 확정
먼저 정책을 아래처럼 고정하는 것이 좋다.

### 정책 A
- `USENET_GROUP`
  - `USENET` = 상위권한
  - `YEONSU` = 하위지점
- `YEONSU`가 등록/수정한 데이터는 같은 Tenant DB에 저장
- `USENET`은 조회 가능
- `YEONSU`는 자기 지점 데이터만 조회

### 정책 B
- `ITWORLD`는 별도 DB
- 기본적으로 `USENET_GROUP`과 연동 안 함

이 정책이 먼저 확정돼야 코드가 단순해진다.

### 계정 예시

- `yeonsu_user`
- `itw_admin`

### 시나리오
1. 연수구 계정이 거래처 추가
   - 연수구 화면에서 보임
   - 유즈넷 화면에서 보임
   - 아이티월드 화면에서는 안 보임
2. 연수구 계정이 거래처 수정
   - 유즈넷에서 수정사항 반영 확인
3. 연수구 계정이 타 지점 거래처 검색
   - 안 나와야 함
4. 유즈넷 계정이 연수구 거래처 조회
   - 나와야 함
5. 아이티월드 계정 로그인
   - 별도 DB 데이터만 보여야 함
6. 집계 화면
   - 연수구는 자기 지점만
   - 유즈넷은 연수구 포함 가능

## 최종 권장안
지금 가장 먼저 구현해야 하는 핵심은 아래 3개다.
1. `TenantCode` 추가
2. `USENET_GROUP` / `ITWORLD` DB 분리
3. `YEONSU -> USENET` 상위조회 정책 반영

이 3개가 들어가면 사용자가 말한 핵심 요구:
- 연수구가 거래처 추가/수정하면 유즈넷에서도 보임
- 연수구는 자기 지점만 조회
- 아이티월드는 별도 DB
를 먼저 충족할 수 있다.
