# 운영용 실계정 정리 및 권한표 (2026-03-20)

## 목적
- 운영에서 실제로 사용할 계정을 **유즈넷 / 연수구 / 아이티월드** 기준으로 정리한다.
- 현재 프로젝트의 `TenantCode / OfficeCode / ScopeType` 구조에 맞춰 권한을 표준화한다.
- 연수구가 등록/수정한 거래처가 유즈넷 상위권한에서 조회되는 규칙을 운영 기준으로 고정한다.
- 아이티월드는 별도 업무 DB를 사용하는 전제로 운영 계정을 정리한다.

---

## 현재 프로젝트 기준 권한 축

### 1. TenantCode
- `USENET_GROUP` : 유즈넷 + 연수구 공용 권역
- `ITWORLD` : 아이티월드 전용 권역

### 2. OfficeCode
- `USENET` : 유즈넷 본사/상위 권한 지점
- `YEONSU` : 연수구 지점
- `ITWORLD` : 아이티월드 지점

### 3. ScopeType
- `Admin` : 전체 관리 권한
- `TenantAll` : 같은 Tenant 안의 허용 지점을 모두 조회/관리
- `OfficeOnly` : 자기 지점 범위만 조회/관리

---

## 운영 권장 원칙
1. **일반 업무용 계정**과 **시스템 관리자 계정**을 분리한다.
2. 유즈넷/연수구는 같은 Tenant(`USENET_GROUP`)를 사용한다.
3. 연수구가 추가/수정한 거래처는 유즈넷 상위권한 계정에서 조회 가능해야 한다.
4. 연수구 계정은 자기 지점(`YEONSU`) 거래처/거래내역만 조회한다.
5. 아이티월드는 `ITWORLD` 별도 업무 DB를 사용한다.
6. 초기 seed 계정(`admin`, `user`, `itw`)은 운영 전환 후 비활성화하거나 비밀번호를 강하게 교체한다.

## 권한표

### 1. 거래처 / 거래내역 / 집계 / 렌탈 / 납품 기준 권한표
| 기능 | platform_admin | usenet_admin | yeonsu_user | itworld_admin |
|---|---|---:|---:|---:|
| 유즈넷 거래처 조회 | 가능 | 가능 | 불가 | 불가 |
| 연수구 거래처 조회 | 가능 | 가능 | 가능 | 불가 |
| 아이티월드 거래처 조회 | 가능* | 불가 | 불가 | 가능 |
| 유즈넷 거래처 등록/수정 | 가능 | 가능 | 불가 | 불가 |
| 연수구 거래처 등록/수정 | 가능 | 가능 | 가능 | 불가 |
| 아이티월드 거래처 등록/수정 | 가능* | 불가 | 불가 | 가능 |
| 거래처 삭제 | 가능 | 정책에 따라 제한 권장 | 불가 권장 | 정책에 따라 제한 권장 |
| 전표/거래내역 조회 | 가능 | `USENET + YEONSU` | `YEONSU`만 | `ITWORLD`만 |
| 자료 기간별 집계 | 가능 | `USENET + YEONSU` | `YEONSU`만 | `ITWORLD`만 |
| 납품내역 조회 | 가능 | `USENET + YEONSU` | `YEONSU`만 | `ITWORLD`만 |
| 렌탈/설치현황 조회 | 가능 | `USENET + YEONSU` | `YEONSU`만 | `ITWORLD`만 |
| 렌탈 신규 등록/문서작성 | 가능 | 가능 | 가능(자기 지점만) | 가능 |

> `가능*` : `platform_admin`은 전역 관리자이므로 가능하지만, 아이티월드 업무는 운영 사고 방지를 위해 필요 시에만 `ITWORLD` 스코프로 접속하는 것을 권장한다.

## 운영 계정 발급 기준

### 필수 원칙
1. 비밀번호는 문서에 저장하지 않는다.
2. 운영 비밀번호는 `ops\.env` 또는 안전한 비밀 저장소에서 관리한다.
3. 개인별 계정이 필요한 경우 아래 규칙으로 추가 발급한다.

### 아이디 규칙 예시
- 유즈넷: `usenet.<이름>` 또는 `usenet_user01`
- 연수구: `yeonsu.<이름>` 또는 `yeonsu_user01`
- 아이티월드: `itworld.<이름>` 또는 `itw_user01`

### 신규 계정 생성 규칙
- 유즈넷 직원: `TenantCode=USENET_GROUP`, `OfficeCode=USENET`, `ScopeType=TenantAll` 또는 `OfficeOnly`
- 연수구 직원: `TenantCode=USENET_GROUP`, `OfficeCode=YEONSU`, `ScopeType=OfficeOnly`
- 아이티월드 직원: `TenantCode=ITWORLD`, `OfficeCode=ITWORLD`, `ScopeType=TenantAll` 또는 `OfficeOnly`

## 추천 운영 결론
### 최소 운영 세트
1. `platform_admin` : 시스템 운영 전용
2. `usenet_admin` : 유즈넷 상위업무 전용
3. `yeonsu_user` : 연수구 업무 전용
4. `itworld_admin` : 아이티월드 업무 전용

### 핵심 규칙
- 연수구가 거래처를 추가/수정하면 유즈넷에서 조회 가능
- 연수구는 자기 지점만 조회
- 아이티월드는 별도 DB로 분리
- 환경설정의 업체/데이터 권한은 시스템 관리자만 수정
