# 업체 데이터 연동 설정 화면 설계안 (2026-03-20)

## 목적
- 관리자만 업체/지점/연동 정책을 설정할 수 있게 한다.
- 일반 사용자는 본인 권한 범위에 맞는 데이터만 조회/입력한다.
- 유즈넷/연수구는 같은 업체 그룹으로 묶고, 아이티월드는 별도 데이터베이스로 분리한다.
- 연수구가 거래처를 추가/수정하면 유즈넷 상급권한 계정에서 바로 조회 가능하게 한다.
- 업체별로 연동 허용/차단 정책을 환경설정에서 제어할 수 있게 한다.

## 화면 1: 환경설정 > 업체/지점 구조

### 화면 목표
- 어떤 업체(테넌트)가 어떤 DB를 쓰는지
- 그 업체 안에 어떤 지점이 있는지
- 어느 지점이 상위/하위 관계인지
를 관리자 입장에서 한 번에 관리

### 권장 섹션 순서
1. 업체 목록
2. 선택 업체 상세
3. 지점 목록
4. 선택 지점 상세
5. 저장/삭제/연결 테스트 버튼

### 좌측 패널: 업체 목록
컬럼 예시:
- 업체코드
- 업체명
- 저장방식
  - 공용 그룹 DB
  - 독립 DB
- 기본상태
  - 사용
  - 사용안함

예시 행:
- `USENET_GROUP | 유즈넷 그룹 | 공용 그룹 DB | 사용`
- `ITWORLD | 아이티월드 | 독립 DB | 사용`

### 우측 상단: 업체 상세
필드 예시:
- 업체코드
- 업체명
- 저장방식
- DB 연결키
- 상위업체코드(선택)
- 설명
- 사용여부

예시:
- 업체코드: `USENET_GROUP`
- 저장방식: `공용 그룹 DB`
- DB 연결키: `TenantDb_UsenetGroup`

### 우측 하단: 지점 목록 + 편집 영역
컬럼 예시:
- 지점코드
- 지점명
- 소속업체
- 상위지점코드
- 조회범위정책
- 활성여부

예시:
- `USENET | 유즈넷 | USENET_GROUP | (없음) | TenantAll`
- `YEONSU | 연수구 | USENET_GROUP | USENET | OfficeOnly`
- `ITWORLD | 아이티월드 | ITWORLD | (없음) | TenantAll`

### 지점 상세 필드
- 지점코드
- 지점명
- 소속업체
- 상위지점
- 기본권한범위
  - `OfficeOnly`
  - `ParentReadable`
  - `TenantAll`
- 기본창고
- 활성여부

### 이 화면의 핵심 규칙
- `USENET_GROUP` 아래에 `USENET`, `YEONSU`를 둔다.
- `YEONSU`의 상위지점은 `USENET`으로 둔다.
- `ITWORLD`는 별도 업체라 다른 업체와 분리한다.

## 화면 2: 환경설정 > 업체 데이터 연동

### 화면 목표
- 업체/지점 사이에 어떤 데이터가 공유되는지 관리
- 기본은 차단, 필요한 공유만 허용
- 거래처/전표/수금/계약서/집계를 항목별로 조절

### 권장 섹션 순서
1. 연동 정책 목록
2. 정책 상세 편집
3. 정책 설명 / 적용 범위 안내

### 정책 목록 컬럼
- 출발업체/지점
- 대상업체/지점
- 거래처 연동
- 전표 연동
- 수금/지급 연동
- 계약서 연동
- 집계 포함
- 상태

예시 정책
1. `YEONSU -> USENET`
   - 거래처: 예
   - 전표: 예
   - 수금/지급: 예
   - 계약서: 예
   - 집계 포함: 예
2. `ITWORLD -> USENET`
   - 전부 아니오

### 정책 상세 필드
- 출발업체코드
- 출발지점코드(선택)
- 대상업체코드
- 대상지점코드(선택)
- 적용 데이터 종류 체크박스
  - 거래처
  - 거래내역(전표)
  - 수금/지급
  - 계약서
  - 집계
  - 납품내역
  - 렌탈관리
- 읽기만 허용 여부
- 수정 연동 허용 여부
- 양방향 여부
- 사용 여부

### 추천 기본 정책
- `YEONSU -> USENET`: 읽기/조회 연동 허용
- `USENET -> YEONSU`: 연수구 전체목록 강제 노출은 기본 비허용
- `ITWORLD <-> USENET_GROUP`: 전부 비허용

즉:
- 연수구가 거래처를 추가/수정하면 유즈넷은 볼 수 있음
- 연수구는 유즈넷 전체를 자동으로 다 보지 않음
- 아이티월드는 완전히 분리

---

## 사용자관리 화면과의 연결
기존 사용자관리 영역에 아래 필드 추가 권장:
- 소속업체(Tenant)
- 담당지점(Office)
- 권한범위(Scope)
- 연동정책 오버라이드 여부

예시:
- `usenet_admin | USENET_GROUP | USENET | TenantAll`
- `yeonsu_user1 | USENET_GROUP | YEONSU | OfficeOnly`
- `itw_admin | ITWORLD | ITWORLD | TenantAll`

## 권한/표시 규칙
### 관리자
- 업체/지점 구조 탭 보임
- 업체 데이터 연동 탭 보임
- 수정/저장/삭제 가능

### 일반 사용자
- 위 탭 숨김 또는 진입 차단
- 본인 소속업체/담당지점 데이터만 업무화면에서 사용

## UI 원칙
- 현재 환경설정 톤 유지
- 좌측 목록 + 우측 상세 패턴 유지
- 새 색상/새 컴포넌트 최소화
- 사무용 밀도 유지
- 저장/삭제/연결 테스트 버튼 우선순위 분명히

---

## 이 설계가 사용자 요구와 맞는 이유
1. 연수구 -> 유즈넷 연동을 정책으로 명시 가능
2. 아이티월드는 별도 업체/별도 DB로 분리 가능
3. 일반 사용자는 정책을 못 건드리고 관리자만 관리 가능
4. 단순 버튼 숨김이 아니라 서버 정책과 연결 가능한 구조
