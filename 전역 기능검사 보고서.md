## 8. 남은 위험

1. Android stable `0.2.25`는 기존 설치본과 업데이트 호환을 위해 기존 stable `0.2.24`와 동일한 `Android Debug` 인증서로 생성했다. 이는 현재 사용자 업데이트에는 필요하지만 장기 운영 보안 측면에서는 릴리스 인증서 전환 계획이 필요하다.
2. Android `0.2.25`는 기능 정상화와 패키징 안정화를 위해 `-DisableAot`/`-DisableTrimming`으로 생성했다. AOT 활성 probe는 ILLink `Out of memory` 및 AOT response file 오류로 실패했으므로, 성능 빌드는 메모리/툴체인 조건 보강 후 별도 검증이 필요하다.
3. 운영 계정별 scope 누수는 자동 로컬 테스트와 운영 게이트 계정 범위 회귀로 점검했지만, 추가 실사용 계정/회사/담당지점 샘플의 read-only SQL 대조는 남아 있다.
4. WPF DatePicker/알림창은 source guard로 범위를 넓혔고 핵심 결제/전표/재고이동 창 5개는 런타임 캡처로 확인했지만, 전체 화면 기준 UIA/스크린샷 전수 검사는 아직 남아 있다.
5. 장시간 자동 새로고침과 사용 중 선택/입력값 보존은 sync 회귀 테스트로 일부 보장되지만, 실제 장시간 관찰은 남아 있다.
