# 코덱스 레거시 판매관리 기획/진행 통합 문서

## 1. 문서 목적
- 본 문서는 코덱스 레거시 판매관리의 전체 진행사항(요구사항, 구현, 수정 이력, 현재 상태)을 한 곳에서 관리하기 위한 기준 문서입니다.
- 운영/유지보수/추가 개발 시 "무엇이 구현되었고 무엇이 남았는지"를 빠르게 파악하는 용도로 사용합니다.

---

## 2. 프로젝트 개요
- 제품명: 코덱스 레거시 판매관리 (구: 클로드 레거시 판매관리)
- 형태: 오프라인 우선(Offline-first) Windows 데스크톱 ERP
- 대상: 소규모 도소매/렌탈 사업자
- 기술 스택:
  - Desktop: .NET 8 WPF (MVVM)
  - Local DB: SQLite
  - Server: ASP.NET Core API
  - Excel Export: ClosedXML
  - (병행 검토) 외부 리포팅 도구 연동/양식 변환 트랙

---

## 3. 진행 목표(요약)
- 기존 레거시 판매관리 업무 플로우를 코덱스 레거시 판매관리에서 안정적으로 재현
- UI 가독성/작업속도 개선 (키보드 중심 + 다크 테마 유지)
- 인쇄 기능 안정화 (미리보기 → 프린터 선택 → 인쇄)
- 데이터 이관/백업/동기화 안정성 강화
- 원장/집계 출력(엑셀) 자동화

---

## 4. 전체 진행 요약 (현재까지)

### 4.1 브랜딩/테마/공통 UI
- 앱 타이틀/문구를 코덱스 레거시 판매관리 기준으로 통일
- 다크 블루 테마 기반 유지, 텍스트 대비(흰색/검정) 반복 보정
- 입력창 글자색/배경색 정리 (가독성 우선)
- 로그인 화면 크기/배치 개선, 아이디/비밀번호 저장 기능 추가

### 4.2 메인 화면/탭/목록 UX
- 전표 목록 중심 3패널 구조 개선
- 거래처 목록 색상/선택 상태 스타일 정리
- 더블클릭 편집 진입 동작 추가 (전표/거래처)
- 우클릭 메뉴 확장:
  - 거래처 수정/삭제
  - 전표 다중 선택 삭제 + 확인/취소 대화상자
- 일부 불필요 요소 제거(요청 기준)

### 4.3 판매(매출) 창
- 품명 검색 UX 개선:
  - Enter 시 1건 매칭 자동입력
  - 다건 매칭 시 선택창 분기
  - 품명 옆 조회 버튼 + 상품목록 더블클릭 입력
- 인라인 편집/입력 영역 정리
- 닫기 시 자동 저장 흐름 개선
- 인쇄/세금계산서/견적서/대금청구서 출력 옵션 확장

### 4.4 수금/지불 창
- 레이아웃 및 테마 정렬
- 스크린샷 기준 반복 보정
- 메인 흐름과 UI 일관성 확보

### 4.5 거래처/상품 관리
- 거래처 검색창에서 거래처 등록 진입
- 상품목록 창에서 상품 등록 진입
- 전체 거래처 메모 초기화 요청 반영

## 5. 인쇄 시스템 진행 내역

### 5.1 핵심 문제 대응
- 기존 문제:
  - 인쇄 시 빈 PDF 오픈
  - 외부 PDF 뷰어 자동 실행
  - 양식 깨짐/오프셋/선 튀어나옴
- 대응 결과:
  - 미리보기 기반 인쇄 흐름 정립
  - 프린터 선택창 기반 인쇄 유지
  - A4 맞춤/정렬 보정 반복 적용

### 5.2 거래명세서
- 1월14일 거래명세서 샘플 기준으로 기본양식 보정
- A4 정렬(가로/세로 중심), 줄/칸 정렬, 선 품질 개선
- 회사설정의 직인 이미지 고정 위치 출력 반영
- PNG 직인 처리 시 배경 투명/테두리 이슈 보정

### 5.3 세금계산서/견적서/대금청구서
- 판매창에서 세금계산서 인쇄 버튼 분리
- 견적서/대금청구서 체크박스 기반 다중 출력 흐름 추가
- 대금청구서 첨부서류 선택 팝업 구현/개선:
  - 체크 순서 번호 부여
  - 거래처별 선택/순서 저장
  - 메인 체크 연동 항목 비활성 처리
  - UI 모던화(리스트/하단 버튼 재배치)

### 5.4 외부 리포팅 도구 트랙
- 양식 폴더 자동 스캔/선택 로더 연결 시도
- .fr3 -> .frx 전환 워크플로우/체크리스트 문서화
- 디자이너 실행/양식 파일 연계 보완
- 최종 운영은 기본양식(WPF 중심) 우선, 외부 리포팅 도구는 병행 유지

## 8. 신규 구현: 자료 기간별 집계 + 4종 원장 엑셀 출력

### 8.1 구현 범위
- 신규 팝업: 자료 기간별 집계
- 4종 원장 출력:
  1) 판매+구매 거래원장
  2) 판매/매출 거래원장
  3) 구매/매입 거래원장
  4) 수금/지불 거래원장
- 저장 후 자동 열기
- A4 가로 1페이지 맞춤 + 헤더 반복 인쇄

### 8.2 반영된 코드(핵심)

- `Desktop/SalesMaster.Desktop.App/MainWindow.xaml.cs`

### 8.3 추가 보정(최근)
- 최근거래일 우선 정렬 적용
- 비고는 전표 비고/메모 우선 사용(거래처 마스터 비고 미혼입)
- 수금/지불 중복 제거 키 보정
- `Transactions` 테이블 누락 DB 대응(자동 생성/보강)
- 집계 팝업 버튼/정렬/UI 계층(Primary/Secondary) 개선

### 8.4 산출 파일 규칙

  - `내문서\SalesDoctor\Exports\`
  - 또는 설정된 ExportPath
- 파일명:
  - `{From}~{To} 의 {원장종류} 거래원장_{yyyyMMdd_HHmmss}.xlsx`
