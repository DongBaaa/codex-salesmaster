# 코덱스 레거시 판매관리 기획/진행 통합 문서

## 1. 문서 목적
- 지금까지 요청/수정/적용한 내용을 한 문서로 통합 관리한다.
- 향후 유지보수 시 "무엇을 왜 바꿨는지" 빠르게 추적할 수 있도록 기준 문서로 사용한다.

## 2. 프로젝트 개요
- 제품명: 코덱스 레거시 판매관리 (구: 클로드 레거시 판매관리)
- 형태: 오프라인 우선 Windows 데스크톱 ERP
- 대상: 소규모 도소매/렌탈 사업자
- 기술:
  - Desktop: .NET 8 WPF (MVVM)
  - Local DB: SQLite
  - Server: ASP.NET Core API

## 3. 핵심 방향
- 다크 블루 테마 유지 + 가독성 개선
- 키보드 중심 업무(F6/F9/F12) 강화
- 전표 단위 데이터 보존/복원 강화
- 인쇄 흐름 안정화 (빈 PDF/외부 자동오픈 문제 제거)

## 4. 현재까지 반영된 주요 변경

### 4.1 앱 명칭/기본 UI 정리
- 타이틀/문구를 "코덱스 레거시 판매관리" 기준으로 정리.
- 다크 테마 유지하면서 텍스트 대비(흰 글자/검은 글자) 문제를 지속적으로 보정.
- 여러 창에서 텍스트박스 입력값 가독성 개선(요청 기준 검은 글자 우선).

### 4.2 메인 화면/목록 UX
- 전표 목록 더블클릭으로 수정 진입 가능하도록 반영.
- 거래처 목록 글자색/선택 시 배경-전경 대비 조정.
- 일부 탭/설명 제거 요청 반영(수금 입력 탭/단축키 설명 등 화면 구성 정리 작업).

### 4.3 판매(매출) 창 UX
- 품명 입력 후 Enter 시:
  - 1건 매칭: 자동 입력
  - 다건 매칭: 선택 창으로 분기
- 품명 옆 조회 버튼 추가 후 상품 목록에서 더블클릭 입력.
- 입력 영역 크기/배치 조정, 인쇄 버튼/옵션 영역 보강.
- 닫기 시 자동저장 처리 개선(저장 실패 시 예/아니오 분기 보완 요청 반영).

### 4.4 수금 입력 창
- 기존 스크린샷 기준으로 레이아웃/색상 보정 반복 적용.
- 메인 테마와 톤 맞추는 방향으로 수정.

### 4.5 거래처/상품 등록 흐름
- 거래처 검색 창에서 거래처 등록 버튼 연동.
- 상품 목록 창에서 상품 등록 버튼 연동.
- 조회/등록/선택 흐름을 팝업 기반으로 단축.

## 5. 인쇄 시스템 개편 (핵심)

### 5.1 기존 문제
- 인쇄 버튼 시 외부 PDF가 먼저 열리고, 빈 PDF 또는 깨진 출력이 발생.
- 사용자 기대 흐름(편집 → 저장 → 미리보기 → 프린터 선택 → 인쇄)과 불일치.

### 5.2 목표 UX (반영)
1. 판매창에서 "출력물 편집" 클릭
2. 편집창에서 거래명세서 데이터 수정 + 미리보기
3. 저장 시 전표 ID 기준 저장
4. 판매창 "인쇄하기" 클릭 시 미리보기 창 먼저 표시
5. 미리보기 창 "인쇄" 클릭 시 Windows PrintDialog 표시
6. 프린터 선택 후 실제 인쇄

### 5.3 적용 설계 (A안)
- PDF 자동 오픈 경로 제거
- WPF 문서 기반으로 전환:
  - `InvoicePrintModel` -> `FixedDocument` 생성
  - `DocumentViewer`로 미리보기
  - `PrintDialog.PrintDocument`로 인쇄

- 효과: 같은 매출 전표를 다시 열어도 편집 내용 유지

## 6. 거래명세서 양식 반영 현황

### 6.1 최근 반영
- 사용자 요청 "1월14일 거래명세서.pdf" 기준으로 레이아웃 재구성.
- `WpfInvoicePrintService`를 교체하여 다음 구성 적용:
  - A4 1페이지 내 상/하 2매 구성
  - 상단(적색): `[공급자 보관용]`
  - 하단(청색): `[공급받는자 보관용]`
  - 공급자/공급받는자 정보 블록
  - 품목 테이블(순번/품명/규격/단위/수량/단가/금액)
  - 합계/입금/미수/인수자 영역

### 6.2 현재 상태
- 기능적으로 미리보기/인쇄 파이프라인은 정상 동작.
- 실물 출력물의 미세 정렬(칸 폭/폰트/줄 높이)은 추가 피드백 기반 2차 보정 예정.

## 7. 외부 리포팅 도구 관련 진행
- 양식 폴더 스캔/템플릿 로더/디자이너 경로 탐지 관련 기능을 단계적으로 붙임.
- `.fr3 -> .frx` 전환을 위한 보조 스크립트/테스트 폴더 운영:
  - `analyze_fr3.ps1`
  - `decode_fr3.ps1`
  - `convert_fr3_utf8.ps1`
  - `_frapi`, `_frnettest`, `_frlegacybridge`, `_frfulltest` 등
- 현재 거래명세서 기본 경로는 WPF 고정문서 인쇄(A안) 중심으로 운영.
- 외부 리포팅 도구는 별도 템플릿/레거시 대응 트랙으로 유지.

## 10. 남은 작업(우선순위)

### High
- 거래명세서 실물 출력물의 원본 양식 정합(칸 단위 픽셀 보정)
- 사용자 인쇄환경(프린터 드라이버별 여백 차이) 대응값 보정

### Medium
- 출력물 편집창에서 수량/단가 입력 시 금액 자동계산 강화
- 편집창의 필드 그룹화/검증 메시지 개선

### Low
- 외부 리포팅 도구 템플릿 편집 UX와 WPF 인쇄 UX 간 통합 옵션 정리
- 인쇄 종류별(거래명 A4/영수증/출고증) 세부 템플릿 확장
